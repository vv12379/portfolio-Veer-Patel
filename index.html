Active Low-Pass Filter Schematic

Second-order Sallen-Key low-pass filter using dual LM741 op-amps. Designed the feedback network with 10k resistors and 100nF capacitors for a 159Hz cutoff frequency. Ran electrical rules check (ERC) with zero errors, assigned SOIC-8 and 0603 footprints, and exported the netlist for PCB layout.

KiCad ERC Netlist Footprint Assignment
PCB

Two-Layer PCB Layout

50mm x 30mm two-layer board layout for the active filter design. Placed components following signal-flow logic, routed all traces manually with 0.25mm signal widths and 0.5mm power rails. Includes ground plane fill, decoupling capacitors near power pins, and proper thermal relief pads. Passed DRC with all clearances met.

LibrePCB Trace Routing DRC Ground Plane
Video

Transient Analysis — RC Filter

SPICE simulation of an RC low-pass filter driven by a 10kHz pulse source. Ran transient analysis over 200 microseconds to verify the exponential charging/discharging behavior and cutoff frequency. Plotted input vs. output voltage waveforms, inductor current, and node voltages to confirm the -3dB point matches the calculated 159Hz.

Ngspice Transient Analysis Waveform Plotting Verification
RF

S-Parameter & Smith Chart Analysis

RF impedance matching network simulation using Qucs-S with Ngspice backend. Analyzed S11 return loss and S21 gain over 1-10 GHz for a 2.4 GHz band-pass filter. Used Smith chart visualization to verify impedance transformation from source to load, confirming less than -15dB return loss at the design frequency.

Qucs-S S-Parameters Smith Chart RF Design

About

Background

EDA and circuit design specialist with hands-on experience in open-source electronic design automation tools. I focus on clean schematic practices, simulation-driven verification, and manufacturable PCB layouts for analog and mixed-signal circuits.

My work spans schematic capture and PCB layout in KiCad and LibrePCB, SPICE simulation and transient analysis in Ngspice, and RF/S-parameter analysis using Qucs-S. I follow a simulation-first methodology — verifying circuit behavior before committing to physical board layouts.

I bring strong attention to detail in component selection, trace routing, and design rule compliance, with a systematic approach to the full schematic-to-fabrication pipeline.
